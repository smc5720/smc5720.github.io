## 3. 액센트 컬러 인터랙티브 가이드

`--color-accent` (`#C8FF00`) 는 다크 배경과 17:1에 가까운 대비를 가지므로 인터랙티브 요소에 폭넓게 사용할 수 있습니다.

### 허용 용도

| 용도 | 구현 패턴 |
|------|-----------|
| 텍스트 링크 hover/active 상태 | `color: var(--color-accent)` on `:hover` / `:focus-visible` |
| 카드 타이틀 hover 강조 | `color: var(--color-accent)` on `.card:hover .post-title` |
| TOC 활성 항목 | `color: var(--color-accent)` + `border-left: 2px solid var(--color-accent)` |
| Reading Progress 바 | `background: var(--color-accent)` (장식 요소 — 텍스트 없음) |
| Primary 버튼 배경 | `background: var(--color-accent)` + `color: var(--color-accent-ink)` (17.0:1 AAA) |
| 포커스 링 | `outline: 2px solid var(--color-accent)` (§4 참조) |
| CategoryBadge 도트 | `background: var(--cat-*)` — 각 카테고리 컬러 적용, 텍스트 없음 |

### 주의 사항

- **`--color-accent` 를 배경으로 쓸 때**: 전경은 반드시 `--color-accent-ink` (`#07070A`) 로 지정합니다 (17.0:1 AAA).
- `color-mix(in srgb, var(--color-accent) N%, transparent)` 로 불투명도를 낮춘 파생 컬러는 대비비가 크게 떨어집니다. 의미 있는 텍스트에는 원색을 사용하고, 소프트 변형은 장식·배경에만 허용합니다.
- 라이트 테마(`[data-theme="light"]`)에서는 `--color-accent-soft` 등 별도 토큰이 지정됩니다 — 라이트 테마 토큰 대비비는 별도 검증 필요.

## 4. 포커스 링 명세

소스: `src/app/globals.css` L116–121

```css
:focus-visible {
  outline: 2px solid var(--color-accent); /* #C8FF00 */
  outline-offset: 2px;
  border-radius: var(--r-xs); /* 4px */
}
```

| 속성 | 값 | 비고 |
|------|----|------|
| 색상 | `var(--color-accent)` = `#C8FF00` | vs `--color-bg` (#07070A): **17.0:1 AAA** |
| 두께 | `2px` | WCAG 2.1 SC 2.4.11 (AA, 2px 이상) 충족 |
| 오프셋 | `2px` | 요소 경계에서 바깥쪽 2px 이격 |
| 반경 | `var(--r-xs)` = `4px` | 요소의 border-radius 와 일치 |
| 트리거 | `:focus-visible` | 마우스 클릭은 링 미표시, 키보드·스위치·보조기기는 표시 |

> 컴포넌트 레벨에서 포커스 링을 재정의할 경우 (`outline: none` 등) 반드시 동등 이상의 대비를 가진 대체 인디케이터(`box-shadow`, `border` 등)를 제공해야 합니다. 현재 codebase에서 재정의 케이스는 없습니다.

## 5. 키보드 내비게이션

현재 사이트는 커스텀 단축키 없이 **표준 브라우저 키보드 내비게이션**을 전면 지원합니다.

### 지원 키

| 키 | 동작 |
|----|------|
| `Tab` | 포커스를 다음 대화형 요소로 이동 |
| `Shift + Tab` | 포커스를 이전 대화형 요소로 이동 |
| `Enter` | 링크 이동 / 버튼 활성화 |
| `Space` | 버튼 활성화 (체크박스·토글 포함) |
| `Esc` | 모달·오버레이 닫기 _(현재 해당 요소 없음)_ |
| 화살표 키 | ARIA 위젯 내부 탐색 _(현재 해당 위젯 없음)_ |

### 포커스 순서 원칙

- DOM 순서 = 시각적 순서 — `tabindex` 양수값 사용 금지.
- Skip-to-content 링크는 미구현 상태입니다. 콘텐츠가 늘어나면 추가를 권장합니다.
- `aria-pressed` 로 토글 상태를 노출하는 컨트롤(카테고리 chip, 정렬 세그먼트)은 현재 구현되어 있습니다.
