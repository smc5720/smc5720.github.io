## 왜 바꿨나

기존 블로그는 **Jekyll + Minimal Mistakes** 조합이었습니다. 테마 자체는 훌륭하지만, 커스터마이징할 때마다 Ruby 생태계와 Liquid 템플릿을 다뤄야 했고, 내가 실제로 작업하는 JavaScript/TypeScript 생태계와 거리가 있었습니다.

결정적으로, 디자인을 완전히 내 것으로 만들고 싶었습니다.

## 새로운 스택

- **Next.js 15** — App Router, 정적 export
- **Tailwind CSS v4** — 새로운 CSS-first 설정
- **MDX** — Markdown에 React 컴포넌트를 섞어 쓸 수 있음
- **Shiki** — 코드 하이라이팅 (tokyo-night 테마)
- **GitHub Actions** — 자동 빌드 & 배포

## 코드 예시

```typescript filename="src/lib/posts.ts"
export function getPostBySlug(slug: string): Post | null {
  const filePath = path.join(POSTS_DIR, `${slug}.mdx`);
  if (!fs.existsSync(filePath)) return null;

  const raw = fs.readFileSync(filePath, "utf-8");
  const { data, content } = matter(raw);

  return {
    slug,
    title: data.title,
    content,
    // ...
  };
}
```

## 디자인 방향

다크 테마를 기본으로, `#C8FF00` 전기 라임을 포인트 컬러로 사용했습니다. Fraunces(가변 광학 세리프)와 Syne(기하학적 산세리프)의 조합으로 에디토리얼한 느낌을 목표로 했습니다.

> 디자인은 기억에 남아야 한다. 그렇지 않으면 존재 이유가 없다.
