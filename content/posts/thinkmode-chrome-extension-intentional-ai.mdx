## 개요

AI가 개발자에게 미치는 영향을 연재해온 저자가 직접 만든 도구를 공개했다. **ThinkMode**는 AI에 프롬프트를 입력하기 전에 사고 모드를 고르고 사용 후 인지 비용을 추적하는 크롬 익스텐션이다.

AI는 생각을 건너뛰게 만들기 쉽다. ThinkMode는 그 흐름에 제동을 걸어 프롬프트 전후로 스스로 생각하는 습관을 만들도록 돕는다. 브라우저 익스텐션이라 일상적인 LLM 채팅 흐름에 자연스럽게 녹아든다.

## 작동 방식

### 프롬프트 전: 사고 모드 선택

프롬프트를 열기 전에 ThinkMode가 먼저 묻는다 — 지금 무엇을 하려고 하는지. 답에 따라 다섯 가지 모드 중 하나를 추천한다:

- **Explore**: 문제를 풀기 전에 먼저 이해하기
- **Challenge**: 기존 계획의 약점 검증하기
- **Decide**: 선택지와 트레이드오프 비교하기
- **Audit**: 품질·정확성·테스트·엣지 케이스 점검하기
- **Reflect**: 방금 한 작업에서 배움 뽑아내기

프롬프트와 작업의 성격을 맞추는 게 목적이다.

### 프롬프트 후: AI 사용 로그

AI를 쓴 뒤에는 어떻게 활용했는지 직접 기록할 수 있다. 사용 방식은 세 가지로 나뉜다:

- **Supportive**: AI가 내 사고를 넓혀주는 경우
- **Mixed**: 시간은 아끼지만 이해를 압축할 수 있는 경우
- **Risky**: AI가 판단을 대신할 수 있는 경우

각 기록은 **인지 비용 미터**에 쌓인다. 위험도가 높은 사용일수록 미터가 빨리 찬다. 미터가 다 차면 ThinkMode가 지원하는 AI 채팅 페이지를 5분간 차단한다.

![ThinkMode 인지 비용 미터 화면](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/d2qzsfjv0ae152wi27n0.png)

## 기술 구조

ThinkMode는 Manifest V3 기반 크롬 익스텐션으로 React + TypeScript + Vite로 만들었다. 구성은 다음과 같다:

- 콘텐츠 스크립트: 지원 AI 채팅 페이지 감지
- 플로팅 버튼: 익스텐션 진입점
- 백그라운드 서비스 워커: 사이드 패널 조율
- React 사이드 패널: 핵심 워크플로
- 공유 TypeScript 모듈: 추천 로직과 프롬프트 생성

현재 ChatGPT, Claude, Gemini 페이지를 지원한다.

**백엔드나 LLM API 호출은 없다.** 대화를 읽거나 페이지 콘텐츠를 긁거나 분석 데이터를 외부로 보내는 일은 하지 않는다. 추천 엔진은 결정론적이며 단순한 키워드 규칙으로 사고 모드를 고른다.

## 설치 방법

[크롬 웹스토어에서 무료로 설치](https://chromewebstore.google.com/detail/nbecnbmefilkkijpldhnbmmbcochhdpo)할 수 있다. 개발자 모드를 켜면 로컬에서 직접 실행할 수도 있다. 소스 코드는 [GitHub에 공개](https://github.com/JulienAvezou/ai-thinking-mode-browser-extension)되어 있어 포크해서 자유롭게 고칠 수 있다.
