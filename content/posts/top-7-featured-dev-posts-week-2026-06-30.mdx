## 개요

DEV 편집팀이 매주 토요일부터 금요일까지 올라온 글 가운데 직접 고른 인기 글 7편을 정리했다. 이번 주 목록은 유독 결이 다양하다. 맥 미니 한 대로 로컬 AI 서버를 꾸린 홈랩 이야기부터, 링크와 버튼의 접근성 차이를 짚은 UI 기본기, 브라우저 안에서만 돌아가는 웹 요약 확장까지 폭이 넓다.

## 이번 주의 7편

### 맥 미니로 만든 AI 홈랩

[글 보러 가기 →](https://dev.to/lordmathis/building-an-ai-homelab-330o)

@lordmathis는 맥 미니 한 대에 로컬 AI 홈랩을 꾸린 과정을 풀어놨다. 직접 만든 추론 관리자와 Mikoshi라는 자체 에이전트 프레임워크까지 얹었다. MCP 클라이언트, 오디오 서비스, 그리고 macOS에서 Docker를 돌릴 때 겪은 특유의 삐걱거림까지 실전에서 얻은 교훈을 나눈다.

### 링크냐 버튼이냐, 그것이 문제

[글 보러 가기 →](https://dev.to/micaavigliano/link-or-button-that-is-the-question-2n13)

@micaavigliano는 링크와 버튼의 의미론적·접근성 차이를 하나씩 뜯어본다. 키보드 조작, 스크린 리더 동작, 흔히 저지르는 잘못된 관행을 다루고, 마지막엔 어떤 상호작용에 어떤 요소를 골라야 할지 판단할 실용 체크리스트로 마무리한다.

### 애플 북스 하이라이트를 꺼내는 CLI

[글 보러 가기 →](https://dev.to/andreykorchak/i-built-a-cli-to-break-my-highlights-out-of-apple-books-51jn)

책벌레를 위한 도구다. @andreykorchak는 애플 북스가 숨겨둔 SQLite 데이터베이스에 갇힌 하이라이트를 꺼내 마크다운으로 내보내는 Go CLI를 만들었다. 여기에 MCP 서버까지 붙여서, AI 비서가 사용자의 독서 메모를 검색하고 엮어낼 수 있게 했다.

### 스마트홈의 아킬레스건, 벽 스위치 되살리기

[글 보러 가기 →](https://dev.to/shalvah/diy-smart-home-project-restoring-the-wall-switch-5d77)

스마트 전구를 쓰다 보면 실수로 물리 스위치를 눌러 전원이 끊기고 네트워크에서 사라지는 순간이 생긴다. @shalvah는 이 고질적인 문제를 정면으로 다룬다. Zigbee 버튼과 스위치 커버를 Home Assistant에 물려 지연 없이 제어하는 구성으로 답을 찾았다.

### 브라우저 안에서 끝내는 웹 페이지 요약 확장

[글 보러 가기 →](https://dev.to/hr21don/coding-an-extension-that-summarises-web-pages-with-html-css-and-js-opm)

@hr21don은 가벼운 TF-IDF 알고리즘만으로 웹 페이지를 요약하는 크롬 확장 만드는 법을 보여준다. 모든 처리가 브라우저 안에서 끝나 프라이버시에 안전하다. API 키도, 외부 서비스도 필요 없다. HTML과 CSS, 순수 JS만 있으면 된다.

### Claude Code 세션 비용을 보여주는 상태 표시줄

[글 보러 가기 →](https://dev.to/ndrone/building-a-custom-status-line-for-claude-code-5822)

Claude Code 세션이 대체 얼마나 비용을 쓰는지 매번 감으로 짐작하기 지쳤던 @ndrone는 직접 상태 표시줄을 만들었다. 비용, 컨텍스트 사용량, 현재 위치를 실시간으로 확인할 수 있다. SDK도 필요 없이, 표준 입력으로 들어온 JSON을 읽는 간단한 셸 스크립트 하나로 돌아간다.

### 하나의 언어로 모든 런타임을, PureScript

[글 보러 가기 →](https://dev.to/0x1/the-ultimate-polymorphism-purescript-as-a-universal-language-5gdi)

@0x1은 PureScript가 비즈니스 로직을 한 줄도 바꾸지 않고 Node의 V8, Erlang의 BEAM, Chez Scheme처럼 전혀 다른 런타임으로 컴파일되는 과정을 살핀다. 예전 Haxe의 "한 번 짜서 어디서나 돌린다"는 시도와 견주면서, PureScript의 순수성이 어떻게 같은 함정을 피하게 해주는지 설명한다.
