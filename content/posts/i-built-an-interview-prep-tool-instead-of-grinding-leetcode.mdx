소프트웨어 엔지니어 인터뷰를 준비하면서 처음에는 다들 그렇듯 LeetCode를 열고 문제를 무작위로 풀기 시작하려 했습니다.

그런데 많은 인터뷰 준비 방법이 무작위 문제 풀기를 권장하는 것 같았지만, 나는 문제를 암기하는 대신 **근본적인 패턴을 이해하는 데 집중**하기로 했습니다.

구조적 회상을 연습하고 모든 것이 어떻게 연결되는지 파악하는 게 핵심이라고 봤습니다. 200개의 문제를 피상적으로 풀기보다 20개의 패턴을 깊이 이해하는 쪽을 택했습니다. 인터뷰 질문 대부분은 결국 같은 근본 아이디어의 변형이기 때문입니다.

### 어떤 도구를 만들었나

그래서 패턴 인식, 템플릿 회상, 트레이드오프를 소리 내어 설명하는 데 집중한 도구를 직접 만들었습니다.

![Image description](https://dev-to-uploads.s3.amazonaws.com/uploads/articles/360ri59iouzfjgv2ywn9.png)

### 주요 기능

앱에서 연습할 수 있는 것들:

- DS&A 패턴 플래시카드
- 문제 → 패턴 퀴즈
- JavaScript 코드 템플릿 채우기
- 복잡도 퀴즈
- 시스템 디자인 치트시트
- 시스템 디자인 플래시카드
- 트레이드오프 퀴즈
- 설명 시나리오(Explain-back scenarios)

### 직접 써보니

실제로 이 도구로 준비하면서 유용함을 느끼고 있습니다. **패턴을 인식하기 시작하면 구현을 추론하는 게 훨씬 쉬워집니다**.

누구나 쓸 수 있도록 Vercel에 배포했고, 로컬 실행도 가능합니다.

데이터 수집이나 저장은 없고 로컬 스토리지만 씁니다. 로그인 없이 바로 연습을 시작할 수 있습니다. 코드는 오픈소스로, 필요에 따라 포크해서 커스터마이즈할 수 있습니다.

- [웹사이트](https://swe-interview-practice-one.vercel.app)
- [저장소(Repo)](https://github.com/JulienAvezou/swe-interview-practice)

### 기술 스택

- Next.js
- TypeScript
- React
- Tailwind CSS
- LocalStorage
- 정적 타입 데이터 파일(Static typed data files)

이 프로젝트는 AI 지원 개발을 실험해보는 기회이기도 했습니다. 일상 업무에 도움이 되는 내부 도구를 만드는 건 AI의 좋은 활용 사례라고 생각합니다.
